## 1. 服务器部署步骤

### 方式一：Git 拉取（推荐）

```bash
# 1. 登录服务器，进入网站目录
cd /www/wwwroot/tuoyue-tech.com

# 2. 如果还没有克隆仓库，先克隆
# git clone https://github.com/WarriorFalcon/erp-admin.git

# 3. 拉取最新代码
cd erp-admin
git pull origin feature/竞祺_选品决策整合

# 4. 安装依赖
npm install

# 5. 构建生产包
npm run build

# 6. 复制 dist 到网站根目录
cp -r dist/* /www/wwwroot/tuoyue-tech.com/

# 7. 确保 Nginx 配置正确（见下方）
```

### 方式二：直接上传打包文件

```bash

# 2. 解压并部署
```

## 2. Nginx 配置

```nginx
server {
    listen 80;
    server_name tuoyue-tech.com;
    
    root /www/wwwroot/tuoyue-tech.com;
    index index.html;
    
    # 前端路由支持（Vue Router History 模式）
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    # API 代理到后端（如果后端在同一服务器）
    location /api/ {
        proxy_pass http://127.0.0.1:8000/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## 6. 常见问题

### 页面刷新 404
确保 Nginx 配置了 `try_files $uri $uri/ /index.html;`

### API 请求失败
- 检查后端服务是否启动
- 检查 `.env` 中的 `VITE_API_BASE_URL` 是否正确
- 检查 Nginx 代理配置

### 静态资源 404
- 检查 `dist` 文件夹是否正确复制到网站根目录
- 检查 Nginx `root` 路径配置
