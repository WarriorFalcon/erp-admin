# 故障应急手册

## 1. API 大面积 5xx

### 现象
- 接口错误率快速上升，前端大量报错。

### 快速排查
1. 访问健康检查：`GET /api/health/`
2. 查看应用日志（API 层异常堆栈）
3. 检查数据库/Redis 是否可连接
4. 查看最近发布与配置变更

### 临时止血
- 限流高风险写接口
- 下线高负载批量任务入口
- 必要时回滚到上一个稳定版本

### 恢复确认
- 5xx 恢复至阈值内
- `/api/health/` 返回 `status=ok`
- 关键接口抽样可用

---

## 2. MySQL 连接异常或慢查询激增

### 现象
- 应用报数据库连接超时，接口 RT 显著升高。

### 快速排查
1. 执行 `python manage.py preflight_check`
2. 检查 MySQL 活跃连接、慢日志
3. 检查读写分离配置是否误指向
4. 检查是否出现大 offset 分页、全表扫描 SQL

### 临时止血
- 暂停重写流量接口（批量写入、同步任务）
- 提升连接池容量（谨慎）
- 将高峰读流量切到只读副本

### 恢复确认
- 连接错误消失
- 慢查询数量回落
- 核心 API P95 恢复

---

## 3. Redis 异常（缓存/幂等/限流失效）

### 现象
- 缓存命中率骤降、请求抖动、幂等重复执行风险上升。

### 快速排查
1. 检查 Redis 可达性与内存使用
2. 检查 key 淘汰策略
3. 检查应用日志中 cache 相关报错

### 临时止血
- 降低非核心缓存 TTL
- 暂停高频写接口
- 必要时启用降级路径（直接 DB，严格限流）

### 恢复确认
- 缓存读写正常
- 幂等命中与限流逻辑恢复

---

## 4. Celery 任务堆积/失败率高

### 现象
- 采集或同步任务延迟严重，dead-letter 增加。

### 快速排查
1. 检查 worker/beat 是否在线
2. 查看队列堆积长度
3. 查看 dead-letter：`GET /api/ops/dead-letter/`
4. 查看重放记录：`GET /api/ops/replay-audit/`

### 临时止血
- 扩容 worker
- 暂停非关键定时任务
- 对可恢复任务做重放

### 恢复确认
- 队列积压下降
- 任务成功率恢复
- dead-letter 增速收敛

---

## 5. OAuth 授权失败或 Token 刷新异常

### 现象
- 平台授权回调失败、token 过期后请求失败。

### 快速排查
1. 检查授权回调参数是否完整（`code/state`）
2. 检查 `FERNET_KEY` 是否正确
3. 检查 token 刷新任务日志
4. 检查 `PlatformToken.expires_at` 是否异常

### 临时止血
- 手工调用 `POST /api/auth/{platform}/refresh/`
- 暂停受影响平台采集任务

### 恢复确认
- Token 刷新成功
- 平台采集恢复

---

## 6. 运维权限异常（无法访问 ops 接口）

### 快速排查
1. 自检：`GET /api/ops/whoami/`
2. 查看名单：`python manage.py list_ops_admin`
3. 同步白名单：`python manage.py sync_ops_admin_from_settings --dry-run`
4. 授权组用户：`python manage.py set_ops_admin --username <u> --action grant`

### 注意事项
- 生产建议优先用组授权（`ops_admin`），白名单用于兜底。

---

## 7. 应急操作命令速查

```bash
# 发布前自检
python manage.py preflight_check

# 运维权限
python manage.py list_ops_admin
python manage.py set_ops_admin --username admin --action grant
python manage.py sync_ops_admin_from_settings --dry-run

# 任务与健康检查
curl "$API_BASE_URL/api/health/"
curl "$API_BASE_URL/api/ops/dead-letter/"
curl "$API_BASE_URL/api/ops/replay-audit/"
```
