## 三、问题诊断

### 3.1 根本原因

**Nginx 反向代理配置问题**

当前配置可能是：
```nginx
# 错误的配置
location / {
    try_files $uri $uri/ /index.html;  # 所有请求都返回前端
}
```

应该改为：
```nginx
# 正确的配置
location /api/ {
    proxy_pass http://127.0.0.1:8000;  # API请求转发到后端
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
}

location /admin/ {
    proxy_pass http://127.0.0.1:8000;  # Django Admin
}

location /swagger/ {
    proxy_pass http://127.0.0.1:8000;  # API文档
}

location /redoc/ {
    proxy_pass http://127.0.0.1:8000;  # API文档
}

location / {
    try_files $uri $uri/ /index.html;  # 其他请求返回前端
}
```

### 3.2 为什么健康检查能工作？

健康检查接口可能被 Nginx 特殊处理，或者后端直接暴露在80端口。

## 五、给嘉瑞的修复清单

### 5.1 紧急修复（必须）

#### 修复1：Nginx反向代理配置
**问题：** API请求被前端拦截
**解决：** 在 nginx.conf 中添加：

```nginx
server {
    listen 80;
    server_name tuoyue-tech.com;

    # API请求转发到Django
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # Django Admin
    location /admin/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # API文档
    location /swagger/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
    }

    location /redoc/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
    }

    # 前端页面
    location / {
        root /path/to/frontend/dist;
        try_files $uri $uri/ /index.html;
    }
}
```

#### 修复2：Django CORS配置
**问题：** 前端跨域访问会被阻止
**解决：** 在 settings.py 中添加：

```python
INSTALLED_APPS = [
    ...
    'corsheaders',
    ...
]

MIDDLEWARE = [
    'corsheaders.middleware.CorsMiddleware',  # 放在最前面
    'django.middleware.security.SecurityMiddleware',
    ...
]

CORS_ALLOWED_ORIGINS = [
    "https://erp-admin-tau.vercel.app",
    "http://localhost:5173",
]

# 允许携带凭证（cookies）
CORS_ALLOW_CREDENTIALS = True
```

#### 修复3：Django CSRF配置
**问题：** 登录返回403
**解决：**

```python
CSRF_TRUSTED_ORIGINS = [
    "https://erp-admin-tau.vercel.app",
    "http://localhost:5173",
]
```
