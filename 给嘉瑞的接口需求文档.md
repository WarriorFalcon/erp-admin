## 一、订单管理模块

### 1.1 获取订单列表
```
GET /api/orders/
```

**请求参数：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| page | int | 否 | 页码，默认1 |
| page_size | int | 否 | 每页条数，默认20 |
| status | string | 否 | 状态筛选：pending(待审核)/paid(待发货)/exception(物流异常)/completed(已完成) |
| platform | string | 否 | 平台：TikTok/Amazon/Shopee |
| keyword | string | 否 | 关键词搜索（订单号/买家/SKU） |
| time_type | string | 否 | 时间类型：order(下单时间)/paid(付款时间) |
| start_date | string | 否 | 开始日期，格式：2026-04-01 |
| end_date | string | 否 | 结束日期，格式：2026-04-18 |

### 1.2 获取订单状态统计
```
GET /api/orders/status-counts/
```

**响应数据：**
```json
{
  "code": 200,
  "data": {
    "pending": 12,
    "paid": 8,
    "exception": 3,
    "completed": 156
  }
}
```

### 1.3 订单操作
```
POST /api/orders/{id}/confirm/      # 确认订单
POST /api/orders/{id}/ship/         # 发货
POST /api/orders/{id}/cancel/       # 取消订单
POST /api/orders/{id}/remark/       # 添加备注
```

```

```

## 四、通用说明

### 4.1 响应格式统一
```json
{
  "code": 200,          // 200成功，其他为错误码
  "message": "success", // 错误时返回错误信息
  "data": {}            // 业务数据
}
```

### 4.2 分页参数
所有列表接口支持统一分页：
- `page`: 当前页码，从1开始
- `page_size`: 每页条数，默认20，最大100

分页响应：
```json
{
  "total": 100,         // 总条数
  "page": 1,            // 当前页
  "page_size": 20,      // 每页条数
  "results": []         // 数据列表
}
```

### 4.3 认证方式
所有接口需要在 Header 中携带 token：
```
Authorization: Bearer {access_token}
```
