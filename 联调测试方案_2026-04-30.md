## 一、测试环境配置

### 1.1 依赖服务

| 服务 | 版本 | 端口 | 用途 | 状态检查 |
|------|------|------|------|----------|
| Django | 4.x+ | 8000 | 后端 API | `GET /api/health/` → 200 |
| MySQL | 8.0 | 3306 | 主数据库 | 健康检查内含 DB ping |
| Redis | 7.x | 6379 | 缓存/Session/Celery | `redis-cli ping` → PONG |
| Celery Worker | 5.x | - | 异步任务（SMS/库存同步） | `celery -A myproject status` |
| 拓岳AI (New API) | v0.12.1 | 80 | AI 对话/翻译/生成 | `GET http://api.tuoyue-tech.shop/v1/models` → 200 |

### 1.2 环境变量 (.env)

```bash
DEBUG=True                    # 开发模式
DEMO_MODE=false               # 关闭Demo路由，使用正式业务路由
RBAC_ENFORCE=True             # RBAC权限强制
DJANGO_SETTINGS_MODULE=myproject.settings
DJANGO_RUNSERVER_PORT=8000

# 数据库
MYSQL_DATABASE=sku_db
MYSQL_USER=backend
MYSQL_PASSWORD=backend_dev_pass
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306

# Redis
REDIS_URL=redis://127.0.0.1:6379/1
CHANNEL_REDIS_URL=redis://127.0.0.1:6379/2

# SMS
SMS_PROVIDER=aliyun
SMS_PROVIDER_CHAIN=aliyun,tencent
ALIYUN_ACCESS_KEY_ID=your_key
ALIYUN_ACCESS_KEY_SECRET=your_secret
ALIYUN_SMS_SIGN_NAME=拓岳科技
ALIYUN_SMS_TEMPLATE_CODE=SMS_333955505

# AI
TUOYUE_NEW_API_AUTHORIZATION=sk-your_key

# Frontend
VITE_API_BASE_URL=http://127.0.0.1:8000
CORS_ALLOW_ALL_ORIGINS=True
```

### 1.3 数据库准备

```bash
cd backend-new
docker compose up -d          # MySQL + Redis
python manage.py migrate       # 执行迁移
python manage.py create_rbac_groups  # RBAC角色
python manage.py createsuperuser     # 管理员
```

### 1.4 服务启动

```bash
# 终端1：Django
cd backend-new && python manage.py runserver 8000

# 终端2：Celery Worker
cd backend-new && celery -A myproject worker -l info -P solo

# 终端3：Celery Beat
cd backend-new && celery -A myproject beat -l info

# 终端4：前端
cd erp-admin && npm run dev
```

## 四、E2E 跨模块场景

### 场景1：新用户注册到首单上架

```
1. GET  /api/auth/captcha/challenge       → 获取图形验证码
2. POST /api/auth/sms/send-code           → 发送短信
3. POST /api/auth/mobile/login            → 登录/注册 → 得JWT
4. GET  /api/auth/me                      → 验证身份
5. POST /api/goods/                       → 创建商品
6. POST /api/sku/bulk-create/             → 创建SKU
7. POST /api/selection/calculate/         → 选品测算
8. POST /api/ai/generate-title/           → AI标题
9. POST /api/ai/generate-description/     → AI描述
10.GET  /api/goods/?keyword=xxx           → 验证已入库
```

```
1. GET  /api/orders/                      → 查看订单
```

## 五、异常检测指南

### 5.1 接口异常识别

| 异常 | 表现 | 排查方向 |
|------|------|---------|
| 502 | Nginx回502 | 后端进程挂、AI上游不可用 |
| 500 | 服务器错误 | Django日志、异常堆栈 |
| 400 | 参数错误 | 字段名/类型/必填项 |
| 401 | 未认证 | Token过期/缺失 |
| 403 | 无权限 | RBAC角色分配 |
| 404 | 不存在 | ID错误/软删除 |
| 429 | 限流 | SMS/IP频率限制 |

### 5.2 数据一致性验证

| 检测项 | 方法 |
|--------|------|
| 创建后查询不到 | POST创建 → GET查询对比 |
| 更新未生效 | PUT更新 → GET确认 |
| 删除后仍可访问 | DELETE → GET应404 |
| 分页重复/遗漏 | 遍历所有page，去重验证 |
| JWT过期仍可访问 | 等2h后重试 |

### 5.3 性能阈值

| 指标 | 警告阈值 | 严重阈值 |
|------|---------|---------|
| 简单查询 | >500ms | >2s |
| 批量操作 | >5s | >15s |
| AI调用 | >30s | >60s |
